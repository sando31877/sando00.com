대본

): ppt 넘기기


) 안녕하세요. 이번에 네트워크 통신에 대해 발표를 하게 된 강민혁 입니다.

)) 여러분 제가 질문 하나 해볼게요. 여러분들은 혹시 이 세가지 주소의 공통점을 찾으실 수 있나요?
(적당히)
) 바로 주소 앞에 http가 들어간다는 것이죠.
) 그렇다면 혹시 http는 어떤 뜻을 가지고 있을까요? 네트워크 통신 중 가장 대표되는 http 통신에 대해 알아봅시다.

) Hyper Text Transfer Protocol의 줄임말인 http는 직역하자면
) hyper text, 즉 웹문서들을
) transfer, 보내는
) protocol, 형식
) 즉 다시 말해서
) http는 사용자와 서버 사이에서 웹 문서들을 주고받는
) 통신 규약입니다.

) 여기서 한글로는 '통신 규약'
) 흔히 프로토콜이라고 부르는 단어가 나오는데
) 인터넷 상에서는 우리의 계정과 비밀번호, 그리고 영준이가 어젯 밤 봤던 이상한 동영상 기록 등 수많은 개인정보가 돌아다니기 때문에 엄격하게 규정된 통신규약을 따라야 합니다.

) 이러한 통신 규약들에는
) 파일 전송 목적의 FTP
) 이메일 목적의 smtp 등 다양한 통신 규약이 있지만
) http는 이러한 통신 규약 중 가장 대표적인 규약이죠.

)그렇다면 http의 구조는 어떻게 될까요?

) 방금 전에 http는 웹 문서를 주고받는 규약이라고 했죠.
) 만약 우리가 네이버에서 기사 페이지로 이동해달라는 요청을 보내게 되면
) 네이버 서버는 기사 페이지 문서와
) 그 문서를 꾸며주는 디자인 소스,
) 그리고 기능 소스를
) 차례대로 보내게 됩니다.

) 만약 인터넷이 원활하지 않을 때 이러한 요청을 하게 된다면
) 기사페이지 문서만 받고
) 디자인 소스와 기능 소스들을 받지 못해
) 원래는 이러한 화면이 나와야 하지만
) 이러한 화면이 나오게 됩니다. 이런 경험 다들 한번씩 있을 것이라 생각됩니다.

) 여튼 이 모든 것을 http라고 합니다.

) 그런데 만약 http에 문제가 생긴다면 어떻게 될까요?
(응답 행 에러코드 https://hyojin96.tistory.com/entry/HTTP-%ED%86%B5%EC%8B%A0)
예를 한번 들어보죠. '@@아 너 아침 먹었어?' (대답 듣고) '아침을 ~~했구나', ' 그러면 ##아 너 아침 먹었어?' (대답 듣고) '아침 ~~했구나' 만약 이 소통을 웹 문서로 했다면 정상적인 http 통신이 되는겁니다.
이와 동시에 http 통신 성공 코드인 상태코드 200이 나타납니다.
그런데 제가 '민호야 밥 먹었어?, 소영쌤 밥 드셨어요?' 라는 요청이 아닌 '체육쌤 밥 드셨어요?', '국어쌤 밥 드셨어요?' 라는 요청을 했을 때 여기에는 이 요청을 받을 체육쌤 국어쌤이 없기 때문에
) 상태코드 404가 뜨며 통신 오류가 납니다. 우리가 흔히 보는 404 not found가 이에 해당되죠.
)) 이처럼 인증되지 않은 사용자가 요청할 경우상태코드 401
) 사용자가 요청 시간을 초과한 경우 상태코드 408
) 사용자는 문제가 없지만 서버에서 문제가 생겼을 시에는 상태코드 502가 나오는 등 http 요청이 성공적으로 완료 되었는지, 혹은 완료가 안되었다면 어떤 문제가 있는지 알려줍니다.

)이제 여러분들은 누군가 'http가 뭐야?'라고 물어본다면 대답을 할 수 있게 되었습니다.
)하지만 'https가 뭐야?'라는 질문에는 음.. 조금 힘들겠죠.

) 여러분이 인터넷을 할 때 주소를 보면 http가 아닌 https가 있을 때가 있습니다. 그렇다면 https는 무엇일까요?

) https는 secure http를 나타냅니다. 여기서 영어 잘하시는 분 혹시 secure가 어떤 뜻인지 알고 계신가요? 그렇죠. @@@, ###, 안전한 등을 나타내는 단어죠. http에 보안을 추가해 주는 것이 https 인데요.
) 앞에서 http는 웹 문서를 주고받는 통신 규약이라고 했죠. 만약 웹 문서에 우리의 카드 정보같은 개인정보가 있어도 이를 보완화 하지 않고 웹 문서 자체를 서버로 보냅니다.
) 그래서 진우같은 해커가 중간에서
) 이 웹 문서를 빼간다면 사용자의 개인정보를 쉽게 볼 수 있죠.
)이러한 문제점을 보완하기 위해 보완이 된 secure http
) 즉 https 입니다.
)https는 서버와 사용자 사이에 암호화 된 연결 통로를 만들어 그곳에서 웹 문서를 주고 받죠.
) 만약 해커가 개인정보를 빼가려 해도 막히게 되는 거죠.

) 이번 시간에 http에 대해 알아보았는데요. 우리가 인터넷을 하는데 꼭 필요한 http에 대한 설명을 재밌게 들었기를 바라며
) 이번 자유주제 발표 여기서 마치겠습니다. 감사합니다!

느낀점

                느낀점

  네트워크 통신 중 프로토콜, 그 중에서도 http에 대한 다양한 자료 조사를 했다.
  ’네트워크 통신을 하려면 다양한 규약이 있어야되며 이러한 규약을 꼭 지켜져야 한다.’, ‘https는 html 문서를 ssl 등으로 암호하여 http에 보안적 요소를 추가한 것이다.’ 등 다양한 내용을 조사하고 이를 ppt로 만들었다.
  얼마 전까지 통신은 매우 간단하다고 생각했다. 겉으로만 보자면 네트워크 통신은 내가 요청한 것을 서버로 보내고, 그 서버가 나한테 응답하는 것이 끝인 것처럼 보이기 때문이다. 그런데 작년 사회 시간에 경제 부분을 배울 때 네트워크 통신은 간단한 것이 아닐수도 있겠다는 생각이 들었다.
  경제는 겉으로만 보면 물 흐르듯이 흘러가는 것처럼 보이지만 사소한 부분까지 본다면 수없이 많은 이론들과 규칙, 규약들이 얽히고 섥켜 있다. 당시 일반적인 문자 시스템과 아이폰 i메세지 문자 시스템의 장 단점 등을 보고 있던 나는 이러한 수업을 듣고 sms, 문자 송수신에도 이렇게 많은 것들이 있는지 알아보기 시작했지만 별로 알아보지 못하고 끝냈다. 하지만 이번 기회로 네트워크 통신에 대해 어느정도 알게 되었고 발표를 잘 하진 못했지만 다음번에 기회가 된다면 더 조사하고 정리한 후 친구들에게 네트워크 통신에 대해 더 알려주고 싶다.
